More about the input to a function

[ label: subprop] 

  

Let's look at the function G of Example  [FE1] again. We can calculate that G(3)=20. Since 1+2=3, it follows that G(1+2)=20. Since 9=3, it follows that G(9)=20. It is the element x (here 3) of the domain (here R) that is being given as input to G, not the name of the element. It doesn't matter how you represent 3, the value of G at 3 is still 20.

This is summed up by the following theorem:

Proposition: subprop2The Substitution PropertyLet F:A→B be any function, and suppose that aA and a'A. If a=a', then F(a)=F(a').

The last sentence of Specification  [funcspec] can be stated more precisely this way: Proposition: funceqHow to tell if functions are equalIf Fi : Ai → Bi , ( i=1,2), are two functions, then [ label: samefunc]  (F_1=F_2)
((A_1=A_2 ,  B_1=B_2 , ((F_1(x)=F_2(x)))))

Method: funcsameTo show that two functions are the same you have to show they have the same domain, the same codomain and for each element of the domain they have the same value.

Exercise

Suppose F:A→B and G:A→B. What do you have to do to prove that F≠G?

Warning

[ label: funccodw]  Since Formula ( [samefunc] ) is an equivalence, this means that the function S:R→R for which S(x)= x2 is not the same as the function T:R→ R+ for which T(x)= x2 . They have the same domain and the same value at every element of the domain, but they do not have the same codomain. This distinction is often not made in the literature. In some theoretical contexts it is vital to make it, but in others (for example calculus) it makes no difference and is therefore quite rightly ignored. In this text we are purposely making all the distinctions made in a sizeable fraction of the research literature.

 

arity

The arity of a function is the number of arguments taken by the function. The word is most commonly used for symbols denoting functions.

Example

The arity of the function sin is one.

Example

The arity of + is two. It takes two arguments.

A function that takes n inputs is also called a function of n variables. In using the notation given here the order in which the variables are listed is important; for example, one cannot assume in general that f(2,3)=f(3,2).

Remark

A function of two variables may be analyzed as a function f:R×R→R where R×R is the cartesian product of R with itself. In that sense it is a function with one input, which must be an ordered pair. I take that point of view in my class notes []; students in my class from time to time ask me why I don't write f((x,y)).

Remark

One sometimes finds functions with variable arity. For example, one might use MAX for the maximum of a list, and write for example

MAX (9,9,-2,5)=9


Of course, one might take a point of view here analogous to that of Remark * and say that MAX has one input which must be a list.

Example

Computer languages such as Lisp and Mathematica have some functions with variable arity. The expression +(3,5,5) in Lisp evaluates to 13, and so does the expression Plus[3,5,5] in Mathematica.

In general, variable arity is possible only for functions written in prefix or postfix notation with delimiters. When the symbol for addition (and similar symbols) is written in infix, Polish or reverse Polish notation, the symbol must have exactly two arguments. Thus the symbol + in Mathematica has arity fixed at  2. Citations: ArbMan74169, HebMckWea8947, Hat82363.
